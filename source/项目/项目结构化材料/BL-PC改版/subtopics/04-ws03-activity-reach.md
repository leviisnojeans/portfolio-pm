# 工作流03：官方活动触达

> **范围**：平台活动触达机制，资源位系统设计
> **在母话题里的角色**：为业务侧提供可预期的主播触达渠道
> **状态**：🟢 进行中

## 事实

- 现状：活动曝光无专属位，与自然 UI 竞争注意力（来源：源文件1 / 2026-05-21）
- 历史转化率：任务 10%、活动 2%（改版前，来源：源文件3 / 2026-05-21）
- 活动 banner 渗透率改前基线：1%，项目目标：+1pp（来源：源文件3 / 2026-05-21）
- 解法：设计[[官方活动资源位]]系统，在信息架构中给活动一个固定归宿（来源：源文件1+2 / 2026-05-21）
- 主播活动定位为「成长路径一环」，使用 Banner 组件区别于常驻工具
- 活动触达两个关键时机：开播前目标引导、开播后数据复盘时的推荐活动
- 结果：[[活动渗透]] +1.5pp（来源：源文件1+2 / 2026-05-21）

## 判断

- 资源位系统移除了业务侧对临时排布的依赖，创造了可预期的触达渠道
- 历史活动转化率仅 2%，说明触达问题比转化问题更根本

## 决策

- 资源位纳入信息架构治理范畴，而非每次单独运营配置

## 与其他子话题的依赖

- 依赖 02 的模块定位（活动模块归属工具模块）
- 与 03 共享"平台价值承载"的设计原则

## 与母话题的同步点
