# 母话题工作区协议

本目录是一个"母话题"工作区。Claude Code 在这个目录里工作时，必须遵守下面的协议，目的是把发散的多轮对话沉淀成结构化文档，避免子话题之间互相不知道、互相矛盾。

## 目录结构

- `_master.md` — 母话题文档。当前最权威的结论、待办、决策日志。**不允许擅自覆盖**，所有写入必须先给我看 diff。
- `subtopics/` — 子话题文档。每个 `.md` 一个子话题。文件名格式 `NN-slug.md`（如 `02-monetization.md`）。
- `conflicts.md` — 冲突记录。子话题之间、子话题与母话题之间、新旧结论之间的矛盾都登记到这里。
- `CLAUDE.md` — 就是本文件，协议本身。

## 每轮对话协议

### 开始时
1. 读 `_master.md`，了解当前母话题状态、已有子话题列表、未决问题。
2. 扫一遍 `subtopics/` 文件名，知道有哪些子话题在跑。
3. 判断我这轮发言属于哪个子话题：
   - 明确（我说了"继续 02"或上下文清晰）→ 直接进入。
   - 模糊 → **先问我**："这个属于 02-monetization 还是开个新的子话题？" 不要猜。

### 进行中
4. 对应子话题文件用追加方式（不要覆盖历史），按 **事实 / 判断 / 决策 / 待办** 四类记。
	4a. 写入文件时，**重要概念、术语、专有名词用 `[[]]` 框选**（如 `[[转化率]]`、`[[Cloud Party]]`），
	方便 Obsidian 识别为双向链接。判断标准：在母话题或其他子话题里可能被交叉引用的概念都加。
5. 对照其他 subtopic 和 `_master.md`，做冲突核查。**冲突的定义**：
   - 同一个事实判断不同（A 说月活 200w，B 说 150w）
   - 同一个决策方向相反（A 决定做轻量，B 决定做完整）
   - 同一个前提假设不一致（A 假设增长，B 假设下滑）
6. 发现冲突 → 写入 `conflicts.md`，并**在本轮回复里明确提出**，让我决定怎么收敛。**不要替我做决定**。

### 结束前
7. 输出本轮 diff 摘要（不直接写入），格式如下：

   ```
   ## 本轮 diff 提议
   
   ### subtopics/02-monetization.md（追加）
   + 事实：xxx
   + 判断：xxx
   
   ### _master.md（更新）
   ~ 未决问题 #3 改为已解决：xxx
   + 决策日志新增：xxx
   
   ### conflicts.md（新增）
   + 冲突 #N：02 说"主播分成 40%" vs 01 说"主播分成 50%"，需要确认
   ```

8. 等我说 `ok 写入` / `改一下：xxx` / `不写` 再实际落盘。**默认不主动写文件**。

## 切换命令

我会用这些短语切换语境：

| 命令 | 含义 |
|---|---|
| `继续 NN` / `切到 NN` | 切到子话题 NN |
| `开新子话题 X` | 创建 `subtopics/NN-X.md`（NN 取下一个未占用编号），并切过去 |
| `回母话题` | 焦点回到 `_master.md`，常用于整体梳理或决策 |
| `检查冲突` | 主动跑一遍全量冲突核查，输出报告，不做写入 |
| `归档 NN` | 把某子话题标记为已完结，结论汇总进 master |

## 回复格式

每轮回复**顶部加一行状态行**：

```
[母] <母话题名>  [子·当前] <子话题>  [其他] <其他子话题简称, ...>
```

正文回答我的实际问题。结尾按"结束前"的格式输出 diff 提议。

## 边界

- 未经我授权不写任何文件。
- 不要美化冲突——发现就报，哪怕看起来是小事。
- 不主动重组目录或重命名文件。
- 如果我同时打开太多子话题、母话题快撑不住了，主动提示我"现在有 N 个未归档子话题，建议先收一收"。
- `_master.md` 的"决策日志"和"已确认结论"两栏只增不删；要否定旧结论，新增一条标注"推翻 #N"，不要回头改原文。
