# PRD 拆章节目录模式

## 何时使用

- PRD 较长（>3000 字），单文件浏览/审阅吃力
- 多个模块各有独立 owner，分别更新
- 需要把某些章节单独拷给特定干系人 review

## 命名约定

`NN-章节名.md`，NN 是两位数固定编号：

| 文件 | 内容 |
|---|---|
| `00-修订记录.md` | 版本历史 + 重大变更 |
| `01-背景与目标.md` | 业务背景 / 项目目标 / 非目标 |
| `02-用户与场景.md` | 目标用户分群 / 关键场景 |
| `03-范围.md` | In/Out scope |
| `04-功能详述.md` | 核心功能模块（可继续拆 `04-1-xxx.md`） |
| `05-交互与原型.md` | Figma 链接 + 关键 frame |
| `06-数据埋点.md` | 事件 / 指标口径 |
| `07-边界与异常.md` | 异常态 / 合规 / 未成年 |
| `08-上线方案.md` | 灰度 / 监控 / 回滚 |
| `09-风险与依赖.md` | 风险表 + 依赖项 |
| `10-附录.md` | 外链 / 引用 |

## 怎么用

1. 启动项目时**删掉 `prd.md`**（避免和目录模式冲突，Claude 会报错）
2. 参考根目录 `prd.md` 的结构填充每个章节
3. CLAUDE.md 协议会自动识别为目录模式
4. 你说"进 PRD 第 4 章"，Claude 进 `prd/04-功能详述.md`

## 已包含的样例章节

本模板里给了 3 个有内容的章节作为参照：
- `00-修订记录.md`
- `01-背景与目标.md`
- `04-功能详述.md`

其余 8 个章节请按需新建，结构参考 `prd.md` 对应章节。
