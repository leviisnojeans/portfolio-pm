## 1. 背景与目标

### 1.1 业务背景

2026 Q2 数据显示，直播间观众首次打赏率连续 3 个季度下滑（15% → 12%）。初步定位的核心因素：

1. **信息密度过高**：当前礼物面板默认展示 24+ 个礼物，新用户无法快速识别"试水档"。
2. **低价档心智模糊**：现有最低价 ¥6 档（"小心心"）在视觉上与中高价档混排。
3. **主播激励不足**：主播侧无法在房间内实时看到分价格段的收入构成。

参考用研结论：见 [`inputs/user-research.md#2026-MM-DD-礼物面板可用性研究`](inputs/user-research.md)。

### 1.2 项目目标

- **北极星**：观众首次打赏率（7 日内）从 12% → 17%（+5pp）
- **二级**：ARPPU ≥¥38（不下降）；主播次月留存不下降
- **健康度**：客诉率、未成年拦截率不劣化

### 1.3 非目标

本项目**不**做以下事项，相关诉求请去对应项目：
- 主播分成比例调整（→ LIVE-2026-Q3-SHARE）
- 弹幕、抽奖、PK 玩法
- PC / Web 端礼物体验
- 礼物商城内购优化（→ LIVE-2026-Q4-MALL）

## 2. 用户与场景

### 2.1 目标用户

| 分群 | 占比 | 当前行为 | 项目关注度 |
|---|---|---|---|
| 新观众（30 天内首次进任意直播间） | ~22% | 80% 不下滑礼物面板 | 🔴 高 |
| 老观众未打赏 | ~46% | 偶尔点开，未送出 | 🟡 中 |
| 老观众已打赏 | ~32% | 行为稳定 | ⚫ 低 |

### 2.2 关键场景

**场景 A：新观众首次进入直播间，被内容吸引但未到打赏阈值**
- 触发：首次进入任一直播间，停留 >60s
- 当前行为：80% 用户不下滑礼物面板；点开的也无明确目标
- 痛点：信息密度高，看不出"从哪里开始"
- 期望：进入面板 3s 内识别出一个低价试水档位

**场景 B：老观众日常互动，希望低成本表达**
- 触发：常驻直播间，已有互动行为
- 痛点：¥6 起步对临时性表达"过重"
- 期望：有更轻量的表达方式（¥1）但不让自己觉得"廉价"

**场景 C：主播看到房间礼物收入但不知道结构**
- 触发：主播开播中，观察右上角礼物收入
- 痛点：只看到总数，不知道是大额头部还是分散小额
- 期望：分价格段实时呈现，激励主播感谢小额观众

## 4. 功能详述

### 4.1 礼物面板结构改版

**前提**：用户已进入直播间且未折叠互动区。

**核心流程**：
1. 用户点击底栏「礼物」按钮
2. 弹出新版面板：
   - **顶部 Tab**：推荐 / 节日 / 全部 / 我的（4 个，可左右滑）
   - **下方网格**：3 列 × N 行，每个礼物 card 显示图标、名称、价格
   - **底部**：B 币余额 + 充值入口
3. 默认停在「推荐」Tab，按主播等级 + 用户历史动态生成内容
4. 「推荐」Tab 顶部固定一行**试水档区**（¥1 / ¥3 / ¥6 三个 card），视觉上和下方常规礼物明确分层

**字段定义**（前端需要的核心字段）：

| 字段 | 类型 | 来源 | 备注 |
|---|---|---|---|
| `gift_id` | int | 后端 | 唯一 |
| `tier` | enum(1-5) | 后端 | 决定特效等级 |
| `category` | enum | 后端 | starter / normal / festival / mine |
| `price` | int | 后端 | 单位 B 币（1 B 币 = ¥0.01） |
| `display_name` | string | 后端 | 后台可配，最长 6 字 |
| `icon_url` | string | 后端 | 静态图，96×96 |
| `effect_id` | string\|null | 后端 | 关联特效，null 表示无特效 |
| `share_to_creator` | bool | 后端 | 是否参与主播分成 |

**异常态**：
- **网络异常**：本地缓存上次成功加载的面板，顶部 toast「刷新失败，显示缓存」
- **礼物下架**：原位置展示灰态 + 角标"已下架"，点击 toast 提示
- **余额不足**：tap 送出 → 跳充值 sheet（沿用现有 flow）
- **未实名**：tap 送出 → 实名引导 sheet（沿用现有 flow）

### 4.2 新增试水档位与特效

| 档位 | 礼物名 | 价格 | 特效 | 分成 | 备注 |
|---|---|---|---|---|---|
| Tier 1 | "拍拍" | ¥1 | 无特效，仅弹幕区飘字 | ❌ 不参与分成 | 仅平台补贴 |
| Tier 1 | "鼓掌" | ¥3 | 屏幕中部小动效（2s） | ✅ 参与 | 普通分成 |
| Tier 2 | "小奶茶" | ¥6 | 屏幕中部中动效（3s）+ 公屏字条 | ✅ 参与 | 普通分成 |

**关键约束**：
- "拍拍" ¥1 不分成的决策见 [`decisions.md#2`](decisions.md)
- 特效不抢占主播画面中心区（顶部 30% 安全区，底部 25% 安全区）
- 同一观众 30s 内连发 → 特效合并展示，避免刷屏

### 4.3 主播侧房间内收入呈现

**前提**：主播开播中、推流稳定。

**核心改动**：
1. 右上角原"礼物总收入"变为可点开的卡片
2. 点开后弹出半屏 sheet，展示本场直播分段收入：
   - 试水档（¥1-6）：xxx B 币（xx 人次）
   - 常规档（¥7-99）：xxx B 币（xx 人次）
   - 高额档（≥¥100）：xxx B 币（xx 人次）
3. 卡片关闭后回到聚合数字态

**目的**：让主播感知到"小额送礼用户也在贡献"，引导主动互动 / 感谢。

**异常态**：
- 数据延迟 >30s：卡片显示「数据计算中」灰态
- 全场无打赏：卡片不出现（避免负向心理暗示）

## 6. 数据埋点

### 6.1 新增事件

| 事件名 | 触发时机 | 关键属性 |
|---|---|---|
| `gift_panel_open` | 礼物面板首次打开 | `source`（底栏/连击/特效引导）, `room_id`, `tab_default` |
| `gift_starter_view` | 试水档区曝光（>1s） | `room_id`, `gift_ids[]` |
| `gift_tier_view` | 任意礼物 card 曝光（>1s） | `tier`, `gift_id`, `position` |
| `gift_send_attempt` | 点击送出 | `gift_id`, `tier`, `is_first_time_in_room`, `is_first_time_lifetime` |
| `gift_send_success` | 服务端回执成功 | `gift_id`, `tier`, ... |
| `creator_income_card_open` | 主播点开收入卡片 | `room_id`, `live_duration` |

### 6.2 复用事件
- `enter_live_room`、`leave_live_room`、`recharge_*` 沿用现有埋点
- **不要重复埋**：现有 `gift_send` 事件保留，新事件并行上报

### 6.3 北极星指标计算口径

**首次打赏率（7 日内）**：
- 分母：`enter_live_room`（distinct 用户）在 day 0
- 分子：同用户在 day 0-7 内发生 `gift_send_success` 且历史无打赏记录
- 注意：必须在数据团队侧锁口径，避免和现有报表冲突

## 7. 边界与异常

- **未实名用户**：可看面板，点送出 → 实名引导 sheet
- **未成年人**：单笔 ≤¥48，日累计 ≤¥200（沿用平台规则），灰度期保持现规则不变
- **断网 / 弱网**：见 4.1 异常态
- **主播下线中**：礼物面板入口隐藏
- **未成年保护回退**：检测到未成年标签 → 强制隐藏 ¥100+ 档位（即使在「全部」Tab 也隐藏）

## 8. 上线方案

### 8.1 灰度计划

| 阶段 | 时间 | 范围 | 准入指标 |
|---|---|---|---|
| 1 | Week 1 | 白名单 50 主播 + 其房间观众 | 仅做体验验证，不看大盘 |
| 2 | Week 2-3 | 10% 主播 + 全量观众进入这些房间 | 首次打赏率不劣化、客诉率不上升 |
| 3 | Week 4 | 50% | 北极星 +2pp 以上 |
| 4 | Week 5 | 100% | 同上 |

### 8.2 监控指标

- 实时大盘：首次打赏率、ARPPU、客诉率、未成年拦截率
- 报警阈值：客诉率 >0.5‰ 立即报警；ARPPU 周环比 -3% 触发评估
- 复盘节奏：每周一上午由数据同学拉数据，PM 例会同步

### 8.3 回滚预案

- **触发条件**：客诉率 >0.5‰、未成年拦截率显著下降、ARPPU 周环比 -5%、任何 P0 事故
- **操作**：服务端配置开关 → 5min 内关闭新面板，回退至旧 UI
- **负责人**：服务端 <姓名> + Levi
